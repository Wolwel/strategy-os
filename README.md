## Що таке Strategy OS?

Strategy OS — це не просто чат-бот. Це повноцінна **Multi-Agent System**, створена для валідації бізнес-ідей. Коли ви вводите запит, система оркеструє роботу 10-ти спеціалізованих AI-агентів.

**Як це працює:**
1.  **Декомпозиція:** Розкладає ідею на гіпотези для перевірки.
2.  **Роутинг:** Визначає, які експерти потрібні (Маркетолог, Фінансист, Юрист тощо).
3.  **Пошук:** Збирає живі дані з інтернету про конкурентів та тренди (Tavily API).
4.  **Математика:** Розраховує Unit Economics, P&L та точку беззбитковості.
5.  **Синтез:** Формує фінальний звіт зі стратегічними фреймворками (SWOT, Porter's 5 Forces).

## Ключові можливості

- **Розумний Роутинг:** Система економить ресурси, запускаючи тільки потрібних агентів під конкретний запит.
- **Живі дані ринку:** Використовує Tavily API для пошуку актуальної інформації в реальному часі.
- **Фінансове моделювання:** Автоматичний розрахунок CAC, LTV, маржинальності та Burn Rate.
- **Перевірка гіпотез:** Науковий підхід до оцінки ризиків та невизначеності.
- **Адвокат Диявола:** Окремий агент, який критично оцінює ідею та шукає "підводні камені".
- **Сценарне планування:** Генерація Оптимістичного, Реалістичного та Песимістичного сценаріїв розвитку.
- **UI в стилі Linear:** Сучасний інтерфейс на Next.js + Tailwind CSS.

# Технологічний стек та інструменти розробки

Проєкт реалізовано з використанням архітектури монорепозиторію (Monorepo), що об'єднує серверну частину на Python та клієнтський інтерфейс на TypeScript. Нижче наведено детальний опис використаних технологій та бібліотек.

## 1. Бекенд (Backend)

Серверна частина відповідає за бізнес-логіку, оркестрацію AI-агентів, обробку запитів та взаємодію із зовнішніми API.

### Базові технології
* **Python 3.11+**: Основна мова програмування проєкту.
* **FastAPI**: Високопродуктивний асинхронний веб-фреймворк для побудови REST API.
* **Uvicorn**: ASGI-сервер для запуску веб-застосунку.

### Штучний інтелект та робота з даними
* **Groq SDK (AsyncGroq)**: Клієнтська бібліотека для високошвидкісної генерації тексту за допомогою великих мовних моделей (Llama-3-70b / Mixtral-8x7b).
* **Tavily Python SDK**: Інтеграція з пошуковим рушієм, оптимізованим для AI-агентів, для збору актуальних ринкових даних у реальному часі.
* **Pydantic**: Бібліотека для валідації даних та управління налаштуваннями за допомогою анотацій типів Python.

### База даних та конфігурація
* **SQLAlchemy (Async)**: ORM (Object-Relational Mapping) для асинхронної роботи з базою даних.
* **aiosqlite**: Асинхронний драйвер для бази даних SQLite (використовується для середовища розробки).
* **python-dotenv**: Інструмент для завантаження змінних середовища з файлу конфігурації.

## 2. Фронтенд (Frontend)

Клієнтська частина реалізована як сучасний односторінковий застосунок (SPA) з підтримкою серверного рендерингу (SSR) для забезпечення швидкодії та SEO-оптимізації.

### Базові технології
* **Next.js 14**: React-фреймворк, що використовує архітектуру App Router.
* **TypeScript**: Типізована надбудова над JavaScript для підвищення надійності клієнтського коду.
* **Node.js 18+**: Середовище виконання JavaScript.

### Інтерфейс та Стилізація
* **Tailwind CSS**: Utility-first CSS фреймворк для швидкої та гнучкої стилізації компонентів.
* **shadcn/ui**: Набір реюзабельних компонентів інтерфейсу (кнопки, форми, картки), побудований на базі Radix UI.
* **Magic UI**: Бібліотека для реалізації складних візуальних ефектів (наприклад, Bento Grid).
* **Lucide React**: Набір легкових векторних іконок.
* **Framer Motion**: Бібліотека для створення декларативних анімацій та переходів.
* **clsx / tailwind-merge**: Утиліти для динамічного керування класами Tailwind та уникнення конфліктів стилів.

### Взаємодія з API та Рендеринг контенту
* **Axios**: HTTP-клієнт для виконання запитів до бекенду з підтримкою перехоплювачів та автоматичного перетворення JSON.
* **React Markdown**: Компонент для безпечного рендерингу Markdown-контенту, отриманого від AI.
* **remark-gfm**: Плагін для Markdown, що додає підтримку розширеного синтаксису (таблиці, списки задач, автопосилання).
* **@tailwindcss/typography**: Плагін для Tailwind CSS, що забезпечує автоматичне типографічне оформлення згенерованого HTML-контенту.

## 3. Рекомендації щодо масштабування (Future Roadmap)

Для подальшого розвитку проєкту та переходу в стадію Production рекомендується розгляд та впровадження наступних технологій:

### Інфраструктура та Бекенд
* **PostgreSQL**: Заміна файлової бази даних SQLite на потужну клієнт-серверну реляційну СУБД.
* **Redis + Celery**: Впровадження черги повідомлень для асинхронної обробки довготривалих запитів до AI-агентів у фоновому режимі.
* **Docker & Docker Compose**: Контейнеризація застосунку для забезпечення ізоляції середовищ та спрощення розгортання.
* **LangChain / LangGraph**: Використання спеціалізованих фреймворків для оркестрації, якщо логіка взаємодії агентів стане надмірно складною для ручного управління.

### Фронтенд та UI
* **Recharts або Visx**: Бібліотеки для візуалізації даних. Дозволять замінити текстові таблиці на інтерактивні графіки та діаграми.
* **React Query (TanStack Query)**: Для ефективного управління станом сервера, кешування запитів та оптимістичного оновлення інтерфейсу.
* **Zod**: Для валідації форм на стороні клієнта та синхронізації схем валідації з бекендом.
* **Sentry**: Для моніторингу помилок та відстеження продуктивності застосунку в реальному часі.
